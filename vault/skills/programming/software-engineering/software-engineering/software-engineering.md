# Software Engineering

Explaining and applying the core objectives, activities, and practices of software engineering as a discipline, including its role in modeling, problem-solving, and knowledge acquisition. Covers software development lifecycle models (Waterfall, Incremental, Reuse-oriented) and Agile frameworks (Scrum, Kanban, eXtreme Programming), along with principles of Continuous Integration and Continuous Deployment (CI/CD). Includes requirements elicitation techniques such as identifying actors, defining scenarios, and constructing Use Case diagrams. Emphasizes system design principles such as abstraction, modularity, and low coupling, and explores modern architectural styles including Model-View-Controller (MVC), microservices, and component-based architectures.
