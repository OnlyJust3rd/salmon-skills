# L2-compare-system-design-tradeoffs

Learner can compare system design alternatives and explain the tradeoffs between consistency, availability, latency, throughput, cost, and operational complexity.

## Hirable Usage

This is hirable at mid level when a candidate can explain what improves and what worsens when choosing SQL vs NoSQL, write-through vs write-back, strong vs eventual consistency, or hash partitioning vs directory partitioning.
