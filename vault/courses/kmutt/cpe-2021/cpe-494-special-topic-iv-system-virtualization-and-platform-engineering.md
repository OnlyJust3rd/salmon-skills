# CPE 494 - Special Topic IV: System Virtualization and Platform Engineering

This course focuses on system virtualization and platform engineering concepts used in modern computing environments. Students learn to build, deploy, and manage virtualized and cloud-based systems, applying automation and platform management techniques to achieve scalability, reliability, and high availability.

## Course Competencies

- Explain the foundational concepts of Linux system administration, including the file system stack, acyclic graph directory structures, and file attributes such as permissions and ownership.
- Utilize fundamental Linux commands for file and directory management, text processing, and the Vi editor.
- Apply system thinking to maintain system availability, predictability, and recoverability, including user and group management, network configuration using Netplan, and Linux boot process and systemd service management.
- Monitor system performance and manage jobs and processes effectively.
- Schedule cron jobs for automated tasks.
- Manage software packages using Linux package management tools.
- Describe how traditional OS concepts (CPU, memory, disk, network) are mapped to virtual environments.
- Differentiate between hardware virtualization (virtual machines) and OS-level virtualization (containers).
- Compare Type-1 bare-metal hypervisors (e.g., Proxmox, VMware ESXi) versus Type-2 hosted hypervisors (e.g., VMware Workstation).
- Explain the concept of containers for packaging applications and their dependencies for consistent behavior across different environments.
- Describe the Linux kernel mechanics and the layered structure of Docker images.
- Create Dockerfiles to build container images.
- Manage the container lifecycle.
- Utilize Docker volumes for data persistence.
- Explain the architectural shift from monoliths to microservices.
- Deploy and manage containerized applications at scale using Kubernetes.
- Configure Kubernetes components, including Pods, Deployments, StatefulSets, Jobs, CronJobs, Secrets, Resource Requests/Limits, RBAC, and Namespaces.
- Utilize Helm charts for packaging and deploying Kubernetes applications.
- Apply YAML configuration rules for Kubernetes resources.
- Analyze the financial and operational paradigm shifts of cloud computing, including the move from CAPEX to OPEX.
- Differentiate between cloud service models (IaaS, PaaS, SaaS, FaaS) and cloud deployment models (Public, Private, Hybrid).
- Explain cloud pricing structures (On-Demand, Reserved, Spot).
- Apply FinOps principles and cost optimization strategies in cloud environments.
- Implement resource tagging for cost management and governance.
- Evaluate architectural decisions for cloud infrastructure, including regions, availability zones, compute instance families, and storage provisioning types.
- Configure cloud networking, including virtual networks, subnet segmentation, Network Security Groups, and load balancing.
- Manage Cloud IAM (Identity and Access Management) for secure access control.
- Implement VM Scale Sets for horizontal scaling of virtual machines.
- Enforce governance policies using Azure Policy.
- Describe the serverless execution model and its benefits.
- Explain event-driven architectures and the function execution lifecycle, including cold starts.
- Configure various function triggers (HTTP, Timer, Queue, Blob) and output bindings for serverless functions.
- Implement CI/CD pipelines for serverless deployments.
- Define infrastructure through code using Infrastructure as Code (IaC) principles.
- Contrast declarative and imperative automation approaches.
- Utilize Terraform for infrastructure provisioning, managing its state and ensuring idempotency.
- Apply Terraform's core building blocks, including providers, resources, variables, and outputs.
- Use Terraform data sources to retrieve information.
- Implement infrastructure provisioning using Azure Bicep.
- Configure continuous deployment workflows using GitOps.
- Track key infrastructure metrics, including CPU, memory, disk I/O, and network throughput.
- Monitor the 'Golden Signals' of system performance: Latency, Traffic, Errors, and Saturation.
- Design and implement open-source monitoring architectures using tools like Prometheus and Grafana.
- Configure rule-based alerting for infrastructure events.
- Apply project management concepts and practices tailored to IT and platform engineering environments.
