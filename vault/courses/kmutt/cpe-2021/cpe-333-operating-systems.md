# CPE 333 - Operating Systems

CPE 333: Operating Systems covers the theoretical aspects of operating systems, including memory management, process management, I/O management, and information management. Students will explore issues related to Job Control Language, Assembler, Loader, and Link, alongside example cases of operating systems, compilers, interpreters, and utilities. The course is structured around three core themes: Virtualization (Processor and Memory), Concurrency (Thread APIs, Locks, Semaphores, and Deadlock), and Persistence (I/O Systems, File Systems, and Mass-Storage Management). By the end of the course, students will deeply understand the concept of operating systems and be able to comprehend small, basic Linux kernel code.

## Course Competencies

- Explain the theoretical aspects of operating systems, including memory management, process management, I/O management, and information management.
- Discuss issues related to Job Control Language, Assembler, Loader, and Link.
- Analyze example cases of operating systems, compilers, interpreters, and utilities.
- Describe the core software responsible for making programs run, sharing memory, and enabling device interaction.
- Differentiate between a program and a process, and explore process states and the Process Control Block (PCB).
- Understand UNIX process APIs such as fork(), wait(), and exec().
- Evaluate CPU scheduling algorithms including First In, First Out (FIFO), Shortest Job First (SJF), Shortest Time-to-Completion First (STCF), Multi-Level Feedback Queue (MLFQ), and Lottery Scheduling.
- Explain memory virtualization concepts, including physical vs. logical addresses, address translation, and contiguous memory allocation techniques.
- Compare and contrast segmentation, paging, and hybrid schemes for non-contiguous memory allocation.
- Describe how the OS creates the illusion of large virtual memory using backing storage and explain page fault handling and page replacement policies.
- Identify the causes of race conditions and the need for mutual exclusion in multi-threaded processes.
- Evaluate lock mechanisms for protecting critical sections, including Test-and-Set, Compare-and-Swap, and Fetch-and-Add.
- Utilize condition variables and semaphores for thread synchronization, including solving the Producer/Consumer bounded-buffer problem.
- Analyze resource allocation graphs and the conditions leading to deadlock, and discuss deadlock prevention, avoidance, and detection/recovery strategies.
- Explain how I/O is integrated into a system via hierarchical bus structures and study hard disk drive geometry and disk scheduling algorithms.
- Describe the abstraction of the file system and data access methods, and analyze implementation details of file allocation and journaling file systems.
- Comprehend small, basic Linux kernel code.
