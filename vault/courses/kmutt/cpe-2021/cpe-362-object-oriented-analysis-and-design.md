# CPE362 - Object Oriented Analysis and Design

Study of Object-Oriented Analysis and Design (OOAD) for modeling and developing complex software systems based on real-world problem domains. The course covers Object-Oriented Programming and Programming Principles, including abstraction, encapsulation, inheritance, polymorphism, modularity, and separation of concerns, as a foundation for system modeling and implementation.

Students perform object-oriented analysis by identifying domain entities, relationships, and system requirements, and translating them into structured models. The course emphasizes Software Design Visualization using UML class and state diagrams to represent system structure and behavior.

The course focuses on object-oriented design, including responsibility assignment, modular system decomposition, and interface design. Students apply Design Pattern Application and Refactoring Techniques to improve code quality, maintainability, and scalability.

A key component of the course is the end-to-end development workflow, where students transform analysis models into implementable designs and working systems, iteratively improving them through refactoring.

## Course Competencies

- Apply Object-Oriented Programming and Programming Principles, including abstraction, encapsulation, inheritance, polymorphism, modularity, separation of concerns, and interface and implementation separation.
- Perform object-oriented analysis of real-world systems by identifying domain entities, attributes, relationships, and translating requirements into conceptual models.
- Construct and interpret Software Design Visualization models, including class diagrams for system structure and state diagrams for system behavior.
- Design object-oriented software systems, including responsibility assignment, modular system decomposition, and interface and component interaction design.
- Apply Design Pattern Application to solve recurring design problems by selecting appropriate patterns and evaluating trade-offs in design decisions.
- Apply Refactoring Techniques to improve existing codebases by identifying code smells, performing incremental design improvements, and enhancing readability and maintainability.
- Transform analysis and design models into working implementations, ensuring consistency between models and code, and maintainability and extensibility of the system.
- Design, refactor, and implement a medium-scale software system, demonstrating end-to-end OO analysis and design, application of programming principles, design patterns, and refactoring, and integration of multiple components into a cohesive system.
