# CPE 223 - Computer Architectures

Processor technology, input and output, memory hierarchy, interleaved memory, bus, cache, pipelined architectures, and computer arithmetic. Machine instructions, assembly language programming, microprocessor design and physical control. Communication and control of heterogeneous processors (on-chip, PCI, USB, SPI, CAN, RS-232/422, I2C, one-wire), communication and control of homogeneous processors (multi-core, cluster, GPU), introduction to specialized processors (vector, DSP). Experiments on microcomputer, microprocessor and microcontroller interfacing with physical devices.

## Course Competencies

- Apply number representation formats and evaluate computer arithmetic algorithms, including analyzing addition, subtraction, multiplication, and division.
- Analyze machine instruction sequencing, register transfer notations, and various addressing modes, and write and evaluate assembly language programs to execute specific tasks, differentiating between RISC and CISC architectures.
- Explain the internal single-bus and multi-bus organization of a processor's datapath, and evaluate the execution steps of an instruction and compare control logic implementations.
- Analyze pipelined architectures and their impact on execution speed, identify hazards within a pipeline, and evaluate techniques to resolve them.
- Evaluate the performance and cost trade-offs of the memory hierarchy, explain how virtual memory mapping works, and evaluate cache memory operations.
- Analyze methods of accessing input and output devices, compare synchronous and asynchronous bus protocols, and standard communication interfaces.
- Compare and contrast homogeneous and heterogeneous processor architectures, and differentiate between UMA and NUMA systems, including SMP, Multi-core Processors, MPP, cluster computing, and specialized vector processors.
- Design, implement, and evaluate a simple processor on an emulator, and apply theoretical knowledge to successfully demonstrate microcomputer and microprocessor interfacing with physical devices through a mini-project.
