# CPE 334 - Software Engineering

This course introduces you to the core principles and techniques used to create functionally correct, easy-to-use, robust, reliable, and maintainable software systems. Software engineering is much more than just writing code; it is a fundamental discipline that involves the systematic application of engineering principles. Throughout our time together, we will cover the software development lifecycle (SDLC). You will learn how to tackle complex problems through modeling (focusing on relevant details), problem-solving (experimentation under time and budget constraints), knowledge acquisition, and rationale-driven decision making (capturing the context behind design choices).

## Course Competencies

- Explain the core objectives and activities of software engineering, and define software engineering as a modeling, problem-solving, and knowledge acquisition activity.
- Compare and contrast different plan-driven process models (Waterfall, Incremental, Reuse-oriented) and Agile Software Development frameworks (Scrum, Kanban, eXtreme Programming).
- Elicit software requirements by identifying actors and scenarios, and create Use Case diagrams to map out system functionality.
- Apply system modeling and architectural design principles to achieve high abstraction and low coupling between components.
- Differentiate between software faults, errors, and failures, and explain the fundamentals of software testing.
- Utilize UML (Unified Modeling Language) to document design decisions, including static structural aspects with Class and Component diagrams, and dynamic behavioral aspects with Sequence and Activity diagrams.
- Systematically track and control changes in a software project using Software Configuration Management (SCM) techniques, including managing baselines, version identification, and utilizing version control systems like Git.
- Implement software using object-oriented programming principles (encapsulation, inheritance, SOLID principles) and apply continuous code refactoring techniques to ensure and maintain code quality.
- Differentiate between software validation and verification, and apply automated testing frameworks (such as JUnit for Java) and Test-Driven Development (TDD) to write unit, integration, and system tests.
- Discuss the principles of Continuous Integration and Continuous Deployment (CI/CD) and the importance of frequent software releases.
- Explain professional Quality Assurance (QA) topics, including conducting code reviews, using static analysis tools, and tracking quality assurance metrics.
- Document alternatives considered and justifications behind team decisions using Rationale Management.
- Model hardware configurations and software mappings using UML Deployment Diagrams.
- Evaluate team performance after each Sprint to adapt and improve workflow, reflecting the agile principle of continuous review and improvement.
- Explore specific modern architectural styles, including Model-View-Controller (MVC), microservices, and component-based architectures.
