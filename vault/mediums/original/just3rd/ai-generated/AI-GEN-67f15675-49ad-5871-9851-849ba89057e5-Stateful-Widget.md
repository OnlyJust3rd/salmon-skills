# Stateful Widgets: Widgets That Change Over Time

In Flutter, everything you see on the screen is a widget. We've learned about stateless widgets, which are like a snapshot – they don't change once built. But what happens when you need a widget that can update itself based on user interaction or other events? That's where **Stateful Widgets** come in.

## What is a Stateful Widget?

A Stateful Widget is a widget that can change its appearance or behavior over time. Unlike StatelessWidget, which is immutable, a StatefulWidget has a mutable state that can be modified. This state is what allows the widget to rebuild itself and reflect the changes.

Think of it like this:

*   **StatelessWidget:** A printed photo. Once it's printed, it doesn't change.
*   **StatefulWidget:** A digital display that can update its content.

## The Two Key Parts of a StatefulWidget

A StatefulWidget is composed of two main parts:

1.  **The Widget Itself (StatefulWidget subclass):** This part is immutable and describes how to configure the widget. It holds references to data that *doesn't* change over the lifetime of the widget.
2.  **The State Object (State subclass):** This part is mutable and holds the actual data that can change. It also contains the `build` method, which describes the UI for the current state of the widget.

When the state changes, you call the `setState()` method. This tells Flutter that the state has changed and that the widget needs to be rebuilt. Flutter then calls the `build` method of the State object again, and the UI is updated to reflect the new state.

## When to Use a StatefulWidget?

You should use a StatefulWidget when your widget needs to:

*   **Respond to user input:** For example, a checkbox that toggles its state, a text field that displays user input, or a button that changes its appearance after being tapped.
*   **Update based on asynchronous operations:** Fetching data from a network or timers that trigger changes.
*   **Manage internal data that affects its UI:** Think of animations, scroll positions, or anything that isn't directly passed in as a parameter from a parent widget.

## A Simple Example: A Counter Button

Let's create a simple counter button. When you tap it, the number displayed on the button should increase.

```dart
import 'package:flutter/material.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Stateful Widget Demo',
      home: Scaffold(
        appBar: AppBar(
          title: const Text('Counter App'),
        ),
        body: const Center(
          child: CounterWidget(), // Our StatefulWidget
        ),
      ),
    );
  }
}

// The StatefulWidget itself. It's immutable.
class CounterWidget extends StatefulWidget {
  const CounterWidget({super.key});

  @override
  State<CounterWidget> createState() => _CounterWidgetState();
}

// The State object. It holds the mutable state and the build method.
class _CounterWidgetState extends State<CounterWidget> {
  int _counter = 0; // This is the mutable state

  void _incrementCounter() {
    // Call setState to trigger a rebuild
    setState(() {
      _counter++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      mainAxisAlignment: MainAxisAlignment.center,
      children: <Widget>[
        const Text(
          'You have pushed the button this many times:',
        ),
        Text(
          '$_counter', // Displaying the state
          style: Theme.of(context).textTheme.headlineMedium,
        ),
        ElevatedButton(
          onPressed: _incrementCounter, // Call the method that updates state
          child: const Text('Increment'),
        ),
      ],
    );
  }
}
```

### How It Works:

1.  **`CounterWidget extends StatefulWidget`**: This is our StatefulWidget. It's responsible for creating its associated `State` object. Notice it has no internal variables that change.
2.  **`createState() => _CounterWidgetState()`**: This method is required by `StatefulWidget`. It returns an instance of our `_CounterWidgetState` class.
3.  **`_CounterWidgetState extends State<CounterWidget>`**: This is the actual state object.
    *   **`int _counter = 0;`**: This variable `_counter` holds the state of our widget – the current count. It's initialized to 0.
    *   **`_incrementCounter()`**: This method is called when the button is pressed.
    *   **`setState(() { _counter++; });`**: This is the crucial part. When `setState` is called, Flutter:
        *   Marks this widget as "dirty" (meaning it needs to be rebuilt).
        *   Executes the function passed to `setState` (in this case, incrementing `_counter`).
        *   Schedules a rebuild of the widget.
    *   **`build(BuildContext context)`**: This method is called by Flutter whenever the widget needs to be drawn on the screen. It returns the UI description for the *current* state of the widget. Here, it displays the text and the current value of `_counter`.

When you run this code and tap the "Increment" button:

1.  The `onPressed` callback of the `ElevatedButton` triggers `_incrementCounter`.
2.  `_incrementCounter` calls `setState`.
3.  `_counter` is incremented.
4.  Flutter rebuilds `CounterWidget`.
5.  The `build` method of `_CounterWidgetState` is called again.
6.  The `Text('$_counter')` widget now displays the updated, incremented value.

## Key Takeaways

*   StatefulWidgets are for UI elements that need to change dynamically.
*   They consist of an immutable widget class and a mutable state class.
*   The `setState()` method is essential for triggering UI updates when the state changes.
*   Always place mutable state within the `State` object, not the `StatefulWidget` itself.

By understanding StatefulWidgets, you unlock the ability to create interactive and dynamic user interfaces in your Flutter applications.
