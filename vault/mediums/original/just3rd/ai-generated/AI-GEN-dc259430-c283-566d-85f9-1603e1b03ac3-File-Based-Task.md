# Working with Files in Claude Cowork

Claude Cowork agents can help you manage and interact with your local files and folders. This means the agent can read, write, and organize information directly on your computer, making it a powerful tool for automating tasks that involve your own documents. Understanding how Claude Cowork handles file-based tasks is key to leveraging its capabilities for personal productivity and workflow automation.

## What is a File-Based Task?

A file-based task involves an agent interacting with files and folders stored on your local system. This can include:

*   **Reading:** Opening and processing the content of text files, documents, spreadsheets, etc.
*   **Writing:** Creating new files or appending/modifying existing ones.
*   **Organizing:** Moving, copying, renaming, or deleting files and folders.
*   **Searching:** Finding specific files based on names or content.

This is distinct from tasks that only involve online information or direct conversation. When an agent performs a file-based task, it's working with the data you've provided or created on your own device.

## Practical Scenario: Organizing Project Documents

Imagine you're working on a project and have a folder full of research papers, notes, and draft documents. You want to organize them into subfolders and rename some files for clarity.

**Scenario:** You have a folder named `Project_Alpha` containing several `.txt` files. You want to:
1.  Create a new subfolder called `Raw_Data`.
2.  Move all files that contain the word "dataset" in their name into the `Raw_Data` folder.
3.  Rename any remaining `.txt` files to have a `_final` suffix.

Claude Cowork can be instructed to perform these actions. For example, you might provide instructions like:

"Go to the `Project_Alpha` folder. Create a subfolder named `Raw_Data`. Move all `.txt` files that have 'dataset' in their filename into `Raw_Data`. Then, rename all other `.txt` files by adding `_final` before their extension."

The agent, with the necessary permissions, would then execute these commands on your local file system.

## Practice Task: Automated Note Taker

Let's say you want to use Claude Cowork to automatically save meeting notes.

1.  **Set up:** Create a folder on your computer called `Meeting_Notes`.
2.  **Instruct Claude:** Imagine you've just had a meeting and want to save the discussion points. You would interact with Claude Cowork and provide the content of your notes. Your instruction might be: "Save the following notes as a new text file in the `Meeting_Notes` folder. Name the file `Meeting_2023-10-27.txt`."
3.  **Provide Content:** You then paste or type the meeting notes directly into the prompt.
4.  **Verify:** After Claude confirms the action, navigate to your `Meeting_Notes` folder and check that the file `Meeting_2023-10-27.txt` has been created with the correct content.

You can extend this by asking Claude to categorize notes, create new files based on templates, or even analyze the content of existing notes.
