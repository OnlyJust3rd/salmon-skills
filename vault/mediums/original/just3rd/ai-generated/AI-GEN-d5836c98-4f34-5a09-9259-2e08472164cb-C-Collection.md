# Understanding C# Collections: Lists and Dictionaries

In C# programming, we often need to store and manage groups of related data. Instead of creating individual variables for each item, collections provide efficient ways to handle these data sets. This lesson focuses on two fundamental collection types: `List<T>` and `Dictionary<TKey, TValue>`.

## What are C# Collections?

Collections are data structures that allow you to store, retrieve, and manipulate a group of elements. They are part of the .NET Framework and offer flexibility and performance benefits over using arrays for many scenarios.

### Lists (`List<T>`)

A `List<T>` is a resizable array that stores elements of a specific type. It's ideal when you need an ordered collection where you can easily add, remove, and access elements by their index.

**Key Features:**

*   **Ordered:** Elements maintain their insertion order.
*   **Dynamic Size:** Automatically grows or shrinks as needed.
*   **Indexed Access:** You can access elements using their zero-based index (e.g., `myList[0]`).

### Dictionaries (`Dictionary<TKey, TValue>`)

A `Dictionary<TKey, TValue>` stores key-value pairs. Each key must be unique, and it's used to look up its associated value. This is perfect for scenarios where you need to quickly retrieve data based on an identifier.

**Key Features:**

*   **Key-Value Pairs:** Stores data as pairs, like a real-world dictionary.
*   **Fast Lookups:** Retrieving a value using its key is very efficient.
*   **Unique Keys:** Each key in a dictionary must be distinct.

## Practical Example: Managing Student Grades

Imagine you are building a simple application to manage student grades.

**Scenario:** You need to store a list of student names and their corresponding scores.

```csharp
using System;
using System.Collections.Generic;

public class GradeManager
{
    public static void Main(string[] args)
    {
        // Using a List to store student names
        List<string> studentNames = new List<string>();
        studentNames.Add("Alice");
        studentNames.Add("Bob");
        studentNames.Add("Charlie");

        Console.WriteLine("Student Names:");
        foreach (string name in studentNames)
        {
            Console.WriteLine($"- {name}");
        }

        Console.WriteLine("\n--------------------\n");

        // Using a Dictionary to store student names and their scores
        // Key: Student Name (string), Value: Score (int)
        Dictionary<string, int> studentScores = new Dictionary<string, int>();
        studentScores.Add("Alice", 95);
        studentScores.Add("Bob", 88);
        studentScores.Add("Charlie", 76);

        Console.WriteLine("Student Scores:");
        foreach (KeyValuePair<string, int> entry in studentScores)
        {
            Console.WriteLine($"{entry.Key}: {entry.Value}");
        }

        // Accessing a specific score by name
        Console.WriteLine($"\nBob's score is: {studentScores["Bob"]}");

        // Checking if a student exists
        if (studentScores.ContainsKey("David"))
        {
            Console.WriteLine("David's score found.");
        }
        else
        {
            Console.WriteLine("David is not in the system.");
        }
    }
}
```

In this example:

*   We use a `List<string>` to keep track of student names in the order they were added.
*   We use a `Dictionary<string, int>` to associate each student's name (the key) with their score (the value). This allows us to quickly find a student's score by just providing their name.
