# Reducer Output: Aggregating Your Hadoop Batch Results

In Hadoop batch processing, after the mappers process your raw data, the reducers take over. Their primary job is to aggregate the intermediate data generated by the mappers into the final results. This stage is crucial for summarizing, counting, or combining information to answer your analytical questions.

## What is Reducer Output?

The reducer receives a set of key-value pairs, where the keys are often the same across many pairs (e.g., a word, a product ID, a date). Its task is to process all the values associated with a single key and produce one or more output key-value pairs. This aggregation is what transforms the scattered intermediate data into meaningful batch results.

Think of it like sorting mail. Mappers might sort mail by ZIP code. The reducer then takes all the mail for a specific ZIP code and bundles it up for delivery. In Hadoop, the reducer bundles up all the values related to a particular key.

## Practical Example: Word Count Aggregation

Let's consider the classic word count example.

**Input Data (after Map phase):**
```
(the, 1)
(quick, 1)
(brown, 1)
(fox, 1)
(the, 1)
(lazy, 1)
(dog, 1)
(the, 1)
```

The mapper would have generated pairs like `(word, 1)`. Now, the reducer needs to aggregate these.

**Reducer Logic (Conceptual):**
The reducer sees all the `(the, 1)` pairs. It knows that for the key "the", it has received the value `1` multiple times. Its logic will sum these `1`s.

**Reducer Output:**
The reducer will process each unique key and its associated values. For the key "the", it will sum up all the `1`s it received, resulting in `(the, 3)`.

The final output for this small example would look like:
```
(the, 3)
(quick, 1)
(brown, 1)
(fox, 1)
(lazy, 1)
(dog, 1)
```
This represents the final aggregated count for each word in the input text.

## Practice Task

Imagine you have processed sales data and the mapper has emitted `(product_id, sale_amount)` pairs. Your goal is to find the total sales for each product.

Write pseudocode for a reducer that takes a `product_id` as the key and a list of `sale_amount` values. The reducer should output a single pair: `(product_id, total_sales_for_that_product)`.

**Pseudocode:**

```
function reduce(key: product_id, values: list<sale_amount>):
  total_sales = 0
  for each amount in values:
    total_sales = total_sales + amount
  emit(key, total_sales)
```
