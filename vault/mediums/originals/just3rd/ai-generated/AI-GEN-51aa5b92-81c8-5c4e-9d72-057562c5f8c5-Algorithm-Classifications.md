# Algorithm Classifications

In the world of algorithms, not all algorithms are created equal. Just like tools, some are better suited for specific tasks than others. Understanding how algorithms are classified helps us choose the right one for the job. This lesson explores common ways algorithms are categorized.

## What are Algorithm Classifications?

Algorithm classifications help us group algorithms based on their structural characteristics or their intended purpose. This grouping makes it easier to understand their properties, predict their behavior, and select the most appropriate algorithm for a given problem.

## Common Classifications

Algorithms can be classified in several ways. Two of the most fundamental classifications are:

### By Structure or Design Paradigm

This classification looks at the fundamental approach an algorithm takes to solve a problem.

*   **Iterative Algorithms:** These algorithms use loops to repeat a set of instructions until a condition is met. They typically involve keeping track of state and updating it in each iteration.
*   **Recursive Algorithms:** These algorithms solve a problem by breaking it down into smaller, identical subproblems and calling themselves to solve these subproblems. They require a base case to stop the recursion.

### By Purpose or Problem Type

This classification focuses on the type of problem an algorithm is designed to solve.

*   **Sorting Algorithms:** These algorithms arrange elements of a list or array in a specific order (e.g., ascending or descending). Examples include Bubble Sort, Merge Sort, and Quick Sort.
*   **Searching Algorithms:** These algorithms find the position of a specific element within a data structure. Examples include Linear Search and Binary Search.
*   **Graph Algorithms:** These algorithms operate on graph data structures to solve problems like finding the shortest path or detecting cycles. Examples include Dijkstra's Algorithm and Breadth-First Search (BFS).

## Practical Scenario

Imagine you're organizing a library.

*   **Sorting:** You need to sort books by title, author, or Dewey Decimal System number. Algorithms like Merge Sort are excellent for this, especially for large collections, because they are efficient.
*   **Searching:** A patron asks for a specific book. You'd use a searching algorithm to quickly locate its position on the shelves. If the shelves are organized alphabetically by title, Binary Search would be very efficient.
*   **Graph:** If you wanted to find the quickest route between different sections of the library for restocking, you might think of the library's layout as a graph and use a graph algorithm like Dijkstra's to find the shortest path.
