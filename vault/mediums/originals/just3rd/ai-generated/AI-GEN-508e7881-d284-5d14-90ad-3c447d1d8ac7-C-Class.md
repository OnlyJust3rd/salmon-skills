# Understanding C# Classes

In C#, a **class** is a blueprint for creating objects. It defines the data (fields or properties) and behavior (methods) that objects of that class will have. Think of it like a cookie cutter: the cookie cutter is the class, and the cookies you make are the objects. Classes are fundamental to Object-Oriented Programming (OOP) as they allow us to model real-world entities and create reusable code.

## What is a Class?

A class in C# is a user-defined type that encapsulates data and methods. It acts as a template for creating instances, called objects. Each object created from a class has its own set of data but shares the same behavior defined by the class.

## Practical Example: A `Car` Class

Let's define a simple `Car` class. This class will have properties for the car's make and model, and a method to describe the car.

```csharp
// Define the Car class
public class Car
{
    // Properties (data the car has)
    public string Make { get; set; }
    public string Model { get; set; }

    // Constructor (special method to initialize an object)
    public Car(string make, string model)
    {
        Make = make;
        Model = model;
    }

    // Method (behavior the car can perform)
    public void Describe()
    {
        Console.WriteLine($"This is a {Make} {Model}.");
    }
}
```

In this example:
*   `public class Car` declares our class.
*   `public string Make { get; set; }` and `public string Model { get; set; }` are properties. They hold the state of a `Car` object. The `{ get; set; }` syntax creates automatic getter and setter methods.
*   `public Car(string make, string model)` is a constructor. It's a special method used to create and initialize objects of the class. When you create a new `Car`, you'll provide the make and model here.
*   `public void Describe()` is a method. It defines an action that a `Car` object can perform.

## Creating and Using Objects

Now, let's create objects (instances) from our `Car` class:

```csharp
// In another part of your code (e.g., in your Main method)

// Create a new Car object using the constructor
Car myCar = new Car("Toyota", "Camry");

// Access properties and call methods on the object
Console.WriteLine($"My car is a {myCar.Make}."); // Output: My car is a Toyota.
myCar.Describe(); // Output: This is a Toyota Camry.

Car anotherCar = new Car("Honda", "Civic");
anotherCar.Describe(); // Output: This is a Honda Civic.
```

Here, `myCar` and `anotherCar` are two separate objects, or instances, of the `Car` class. Each has its own `Make` and `Model`, but they both know how to `Describe()` themselves.
