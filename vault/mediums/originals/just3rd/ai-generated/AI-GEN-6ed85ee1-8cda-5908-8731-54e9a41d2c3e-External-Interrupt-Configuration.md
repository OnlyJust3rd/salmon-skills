# External Interrupt Configuration for AVR Microcontrollers

This lesson focuses on the initial setup required to enable external interrupts on AVR microcontrollers. Understanding this configuration is crucial for building responsive embedded systems that react to real-world events. We will explore the specific registers and bit settings that allow your AVR to listen for and respond to signals from external sources.

## What are External Interrupts and Why Configure Them?

External interrupts are signals generated by hardware external to the microcontroller, such as a button press, a sensor detecting a change, or a communication signal. When an external interrupt occurs, the microcontroller temporarily pauses its current task and jumps to a specific piece of code (an interrupt service routine or ISR) designed to handle that particular event.

Configuring external interrupts involves telling the microcontroller:
1.  **Which pins can trigger an interrupt.**
2.  **Under what conditions (rising edge, falling edge, level change) the interrupt should be triggered.**
3.  **Whether the interrupt is enabled or disabled.**

This configuration is the first step before you can actually write the code that runs when an interrupt happens.

## Key Registers for External Interrupt Configuration

AVR microcontrollers use specific hardware registers to manage external interrupts. The most important ones for basic configuration are:

*   **GICR (General Interrupt Control Register):** This register controls the enabling/disabling of specific interrupts.
*   **MCUCR (Microcontroller Control Register):** This register defines the trigger source for the external interrupts (e.g., rising edge, falling edge).
*   **MCUCSR (Microcontroller Control and Status Register):** This register contains bits for more advanced interrupt control, including the trigger for External Interrupt 4 (INT4) on some AVR devices.

Let's look at the relevant bits within these registers.

### GICR - Enabling Interrupts

The GICR typically contains bits like `INT0`, `INT1`, and `INT2` (and potentially `INT3`, `INT4` on more complex AVRs). Setting a bit in GICR to `1` enables the corresponding external interrupt. Setting it to `0` disables it.

| Bit   | Name  | Description                                          |
| :---- | :---- | :--------------------------------------------------- |
| 7     | INT7  | Enable External Interrupt 7                          |
| 6     | INT6  | Enable External Interrupt 6                          |
| 5     | INT5  | Enable External Interrupt 5                          |
| 4     | INT4  | Enable External Interrupt 4                          |
| 3     | INT3  | Enable External Interrupt 3                          |
| 2     | INT2  | Enable External Interrupt 2                          |
| 1     | INT1  | Enable External Interrupt 1                          |
| 0     | INT0  | Enable External Interrupt 0                          |

### MCUCR - Defining Trigger Conditions

The MCUCR allows you to specify *how* an external interrupt is triggered. For `INT0` and `INT1`, you can typically choose between:

*   **Low Level:** Interrupt triggered when the pin is low.
*   **Logical Change:** Interrupt triggered by any change on the pin (from low to high or high to low).
*   **Falling Edge:** Interrupt triggered when the pin transitions from high to low.
*   **Rising Edge:** Interrupt triggered when the pin transitions from low to high.

| Bit | Name | Description                       |
| :-- | :--- | :-------------------------------- |
| 3   | ISC11 | Interrupt Sense Control for INT1  |
| 2   | ISC10 | Interrupt Sense Control for INT1  |
| 1   | ISC01 | Interrupt Sense Control for INT0  |
| 0   | ISC00 | Interrupt Sense Control for INT0  |

The combination of `ISC11` and `ISC10` (and `ISC01`, `ISC00` for `INT0`) determines the trigger:

| ISC11 | ISC10 | Trigger Condition for INT1 |
| :---- | :---- | :------------------------- |
| 0     | 0     | Low Level                  |
| 0     | 1     | Logical Change             |
| 1     | 0     | Falling Edge               |
| 1     | 1     | Rising Edge                |

### MCUCSR - Trigger for INT2 (and higher)

For interrupts like `INT2`, the trigger configuration is often found in the MCUCSR register.

| Bit | Name | Description                       |
| :-- | :--- | :-------------------------------- |
| 6   | ISC21 | Interrupt Sense Control for INT2  |
| 5   | ISC20 | Interrupt Sense Control for INT2  |

The combinations for `ISC21` and `ISC20` are similar to those in MCUCR for triggering interrupt INT2.

## Practical Example: Configuring INT0 for a Button Press

Let's say we want to use pin `PD2` (which is typically assigned to `INT0`) on an ATmega328P (common in Arduino Uno) to detect a button press. We want the interrupt to trigger when the button is *released*, which means we are looking for a **rising edge** (the pin goes from low, pulled by the button to ground, to high when the button is released).

**Steps:**

1.  **Configure the pin as Input:** The pin connected to the button must be set as an input pin in the DDR (Data Direction Register). For `PD2`, this means setting the 2nd bit of DDRD to 0.
2.  **Enable Pull-up Resistor (Optional but Recommended):** To ensure a defined high state when the button is not pressed, it's common to enable the internal pull-up resistor for the pin. For `PD2`, this involves setting the 2nd bit of PORTD to 1.
3.  **Set the Trigger Condition:** We want a rising edge. For `INT0`, this means setting `ISC01` and `ISC00` in `MCUCR` to `1` and `1` respectively.
4.  **Enable the Interrupt:** We need to enable `INT0` in the `GICR` register.

**C Code Snippet (using AVR-GCC syntax):**

```c
#include <avr/io.h> // Include I/O header for register definitions

void setup_external_interrupt0() {
    // 1. Configure PD2 (INT0) as input
    DDRD &= ~(1 << PD2); // Set bit 2 of DDRD to 0 (input)

    // 2. Enable pull-up resistor on PD2 (optional but good practice)
    PORTD |= (1 << PD2); // Set bit 2 of PORTD to 1 (enables pull-up)

    // 3. Set INT0 to trigger on a rising edge
    // ISC01=1, ISC00=1 for rising edge
    MCUCR |= (1 << ISC01) | (1 << ISC00);

    // 4. Enable INT0 interrupt in GICR
    GICR |= (1 << INT0);
}

// Later, in your main function or another setup routine:
// setup_external_interrupt0();
// sei(); // Global interrupt enable (essential for any interrupt to work)
```

In this code:
*   `DDRD &= ~(1 << PD2);` clears the `PD2` bit in `DDRD`, making it an input. The `~` inverts the bitmask `(1 << PD2)` so that only the `PD2` bit is affected.
*   `PORTD |= (1 << PD2);` sets the `PD2` bit in `PORTD` to `1`, enabling the internal pull-up resistor.
*   `MCUCR |= (1 << ISC01) | (1 << ISC00);` sets both `ISC01` and `ISC00` bits in `MCUCR` to `1`, configuring `INT0` for a rising edge trigger.
*   `GICR |= (1 << INT0);` sets the `INT0` bit in `GICR` to `1`, enabling the `INT0` interrupt source.

**Important Note:** After configuring the interrupts, you *must* also enable global interrupts using the `sei()` instruction. This is usually done once after all peripheral setups are complete.
