# SPI Master Data Transfer: Implementing C Code

This lesson focuses on implementing bare-metal C code for SPI master data transfer, a crucial aspect of AVR microcontroller communication programming. You'll learn how to initiate and manage data exchange with an external SPI slave device.

## Understanding SPI Master Mode

SPI (Serial Peripheral Interface) is a synchronous serial communication protocol often used for short-distance communication, especially in embedded systems. As an SPI master, your AVR microcontroller will control the communication bus and dictate when data is sent or received by the slave device.

Key components of SPI communication:

*   **SCK (Serial Clock):** Generated by the master to synchronize data transfer.
*   **MOSI (Master Out, Slave In):** Data line from master to slave.
*   **MISO (Master In, Slave Out):** Data line from slave to master.
*   **SS (Slave Select):** A dedicated line for each slave, asserted (usually low) by the master to select which slave to communicate with.

In master mode, the AVR's SPI hardware allows you to configure the clock polarity, phase, and speed. This enables flexible communication with various SPI slave devices.

## Practical Example: Sending Data to an SPI LCD Display

Let's implement a scenario where an AVR microcontroller acts as an SPI master to send data to a simple SPI-based LCD display. We'll assume the LCD requires a command byte followed by data bytes to display characters.

We'll use the AVR's SPI registers to configure and control the communication.

### Core SPI Registers (AVR ATmega Series Example)

| Register | Description                                                                                                                                                                |
| :------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **SPCR** | **SPI Control Register:** Controls the SPI enable (SPE), master/slave select (MSTR), clock polarity (CPOL), clock phase (CPHA), clock rate (SPR1, SPR0), and SPI interrupt enable (SPIE). |
| **SPSR** | **SPI Status Register:** Contains the SPI interrupt flag (SPIF) which is set when a serial transfer is complete, and the double-speed SPI (SPI2X) bit.                       |
| **SPDR** | **SPI Data Register:** Holds the data to be transmitted or received. Writing to SPDR initiates a transmission. Reading from SPDR retrieves received data.                  |

### Implementing the SPI Master Code

First, we need to set up the SPI pins as outputs for MOSI, SCK, and SS, and an input for MISO. Then, we configure the SPI module in master mode.

```c
#include <avr/io.h>
#include <util/delay.h>

// Define the Slave Select (SS) pin for our LCD
#define LCD_SS_PIN PB0 // Example: Port B, Pin 0

void spi_init_master() {
    // Set MOSI, SCK, and SS as output
    DDRB |= (1 << DDB0) | (1 << DDB1) | (1 << DDB2); // PB0 (SS), PB1 (SCK), PB2 (MOSI) - Assuming these are correct for your target MCU
    // Set MISO as input
    DDRB &= ~(1 << DDB3); // PB3 (MISO)

    // Ensure SS pin is initially high (deselected)
    PORTB |= (1 << LCD_SS_PIN);

    // Enable SPI, set as Master, set clock rate fck/16, CPOL=0, CPHA=0
    SPCR = (1 << SPE) | (1 << MSTR) | (1 << SPR0); // SPE: SPI Enable, MSTR: Master Mode, SPR0: Clock rate fosc/16
    // SPSR |= (1 << SPI2X); // Uncomment for double speed if needed
}

// Function to send a single byte via SPI and return received byte
uint8_t spi_transfer(uint8_t data) {
    // Load the data into the SPI data register
    SPDR = data;

    // Wait for transmission to complete (SPIF flag set)
    while (!(SPSR & (1 << SPIF)));

    // Return the received data
    return SPDR;
}

// Function to select the slave device
void spi_select_slave() {
    PORTB &= ~(1 << LCD_SS_PIN); // Assert SS (pull low)
}

// Function to deselect the slave device
void spi_deselect_slave() {
    PORTB |= (1 << LCD_SS_PIN); // Deassert SS (pull high)
}

// Example function to send a command and data to the LCD
void lcd_send_data(uint8_t cmd, uint8_t data) {
    spi_select_slave(); // Select the LCD

    // Send command byte
    spi_transfer(cmd);
    _delay_us(100); // Small delay between command and data, adjust as needed

    // Send data byte
    spi_transfer(data);
    _delay_us(100); // Wait for data to be processed by LCD

    spi_deselect_slave(); // Deselect the LCD
}

int main(void) {
    spi_init_master();

    // Example: Send command 0x01 (clear display) and data 0x41 ('A')
    // This is a hypothetical example for demonstration.
    // Real LCDs have specific command/data structures.

    // Assume command to send data is 0xF0 and command to send instruction is 0xF1
    // And the actual character to display is 'H' (ASCII 0x48)

    uint8_t instruction_cmd = 0xF1; // Example command for instruction
    uint8_t data_cmd = 0xF0;        // Example command for data
    uint8_t char_H = 0x48;          // ASCII for 'H'
    uint8_t char_i = 0x69;          // ASCII for 'i'

    // Example: Display 'H'
    lcd_send_data(instruction_cmd, char_H);
    _delay_ms(50); // Delay between display updates

    // Example: Display 'i'
    lcd_send_data(instruction_cmd, char_i);
    _delay_ms(50);

    while (1) {
        // Main loop can do other tasks or keep displaying
    }

    return 0;
}
```

### How it Works

1.  **Pin Configuration:** The `DDRB` register is used to set the direction of the SPI pins (MOSI, SCK, SS as output, MISO as input). The `PORTB` register controls the initial state of the SS pin.
2.  **SPI Initialization (`spi_init_master`):**
    *   `SPE` (SPI Enable) is set to turn on the SPI module.
    *   `MSTR` (Master/Slave Select) is set to configure the AVR as a master.
    *   `SPR0` (SPI Clock Rate) is set to `fosc/16`. Other bits (`SPR1`, `SPI2X` in `SPSR`) can be used to select different clock speeds.
    *   `CPOL` (Clock Polarity) and `CPHA` (Clock Phase) are left at their default (0), meaning the clock is idle low and data is sampled on the leading edge. These should be set according to the slave device's datasheet.
3.  **SPI Transfer (`spi_transfer`):**
    *   The `data` byte is written to the `SPDR` register. This automatically starts the serial transfer.
    *   The `while (!(SPSR & (1 << SPIF)));` loop polls the `SPIF` (SPI Interrupt Flag) bit in the `SPSR` register. When `SPIF` becomes 1, it signifies that the transmission (and reception) of the byte is complete.
    *   The received byte is then read from `SPDR` and returned.
4.  **Slave Selection/Deselection (`spi_select_slave`, `spi_deselect_slave`):** These functions control the `SS` pin. Pulling the `SS` pin low selects the slave, and pulling it high deselects it. You must select the correct slave *before* initiating any SPI transfer.
5.  **LCD Send Function (`lcd_send_data`):** This function encapsulates the process of selecting the slave, sending a command byte, sending a data byte, and deselecting the slave. Small delays (`_delay_us`, `_delay_ms`) are often necessary to allow the slave device to process the data. The specific command bytes (e.g., `0xF0`, `0xF1`) and data sent depend entirely on the LCD's datasheet.

## Practice Task

2.  **Configure SPI:** Adapt the `spi_init_master` function for your target AVR microcontroller. Pay close attention to the pin definitions and desired clock speed. Consult the datasheet for the correct bit definitions for `SPCR` and `SPSR`.
3.  **Implement Data Transfer:** Write a function that sends a sequence of bytes to your connected SPI device. For an EEPROM, this might involve sending a read command, an address, and then receiving data. For a sensor, it might involve sending a configuration command and then reading sensor values.
4.  **Verify Communication:** If possible, use a logic analyzer or oscilloscope to monitor the SPI signals (SCK, MOSI, MISO, SS) to verify that the data is being transmitted and received as expected.
