# MongoDB Insert Operations

This lesson focuses on the fundamental `insert` operations in MongoDB, a crucial part of applying CRUD (Create, Read, Update, Delete) operations to manage your data. Understanding how to insert documents is the first step in populating your MongoDB collections.

## What are Insert Operations?

In MongoDB, data is stored in collections, which are groups of documents. Documents are BSON (Binary JSON) objects that represent records. Insert operations are used to add new documents to a collection. MongoDB provides two primary methods for inserting documents:

*   **`insertOne()`**: Inserts a single document into a collection.
*   **`insertMany()`**: Inserts an array of documents into a collection.

## When to Use Insert Operations

You'll use insert operations whenever you need to add new data to your MongoDB database. This could be:

*   Adding a new user to a `users` collection.
*   Recording a new order in an `orders` collection.
*   Storing new product details in a `products` collection.
*   Any time you need to create new records.

## `insertOne()`: Adding a Single Document

The `insertOne()` method is straightforward. It takes a single document as its argument and adds it to the specified collection.

### Syntax

```javascript
db.collection.insertOne(
   <document>,
   <options>
)
```

### Example

Let's say we have a `users` collection and want to add a new user.

**Shell Command:**

```javascript
db.users.insertOne({
  name: "Alice Wonderland",
  age: 30,
  email: "alice@example.com",
  hobbies: ["reading", "gardening"]
})
```

**Explanation:**

*   `db.users`: Refers to the `users` collection in the current database.
*   `insertOne()`: The method to insert a single document.
*   `{ name: "Alice Wonderland", ... }`: This is the document we are inserting. It's a BSON object with key-value pairs representing the user's information.

**Output:**

If successful, the command will return a document similar to this, indicating the acknowledgment of the write operation and the `_id` automatically generated by MongoDB for the new document.

```json
{
  "acknowledged": true,
  "insertedId": ObjectId("60c72b2f9b1e8b0f5a7e3b0c")
}
```

The `_id` field is a unique identifier for each document in a MongoDB collection.

## `insertMany()`: Adding Multiple Documents

The `insertMany()` method is used when you need to add multiple documents to a collection at once. It takes an array of documents as its argument. This is more efficient than calling `insertOne()` multiple times for larger datasets.

### Syntax

```javascript
db.collection.insertMany(
   [ <document1>, <document2>, ... ],
   <options>
)
```

### Example

Let's insert multiple new products into a `products` collection.

**Shell Command:**

```javascript
db.products.insertMany([
  {
    name: "Laptop",
    brand: "TechCo",
    price: 1200,
    inStock: true
  },
  {
    name: "Keyboard",
    brand: "KeyMaster",
    price: 75,
    inStock: true
  },
  {
    name: "Mouse",
    brand: "Clicky",
    price: 25,
    inStock: false
  }
])
```

**Explanation:**

*   `db.products`: Refers to the `products` collection.
*   `insertMany()`: The method to insert an array of documents.
*   `[ { name: "Laptop", ... }, { name: "Keyboard", ... }, ... ]`: This is an array containing three distinct document objects.

**Output:**

The output will confirm the acknowledgment and provide an array of `insertedId`s for each document inserted.

```json
{
  "acknowledged": true,
  "insertedIds": [
    ObjectId("60c72b2f9b1e8b0f5a7e3b0d"),
    ObjectId("60c72b2f9b1e8b0f5a7e3b0e"),
    ObjectId("60c72b2f9b1e8b0f5a7e3b0f")
  ]
}
```

## Important Considerations

*   **`_id` Field**: If you don't explicitly provide an `_id` field for a document, MongoDB automatically generates a unique `ObjectId` for it. You can also provide your own custom `_id` if you have a specific unique identifier you want to use, but it must be unique within the collection.
*   **Duplicate `_id`s**: MongoDB will reject an insert operation if you try to insert a document with an `_id` that already exists in the collection.
*   **Error Handling with `insertMany()`**: By default, if one document in an `insertMany()` operation fails, the entire operation will be stopped, and no documents will be inserted. You can use the `ordered: false` option to allow the operation to continue even if some documents fail to insert.

By mastering these insert operations, you've taken a significant step in managing data within your MongoDB collections.
