# Change Review in Agentic Coding

In agentic coding workflows, effectively reviewing changes generated by an agent is a critical step before integrating them into your codebase. This micro-skill, **Change Review**, focuses on the practice of inspecting generated diffs to ensure accuracy, adherence to requirements, and overall code quality. It's about being the human oversight that catches potential issues before they cause problems.

## Why Change Review Matters

Automated code generation is powerful, but it's not infallible. An agent might misunderstand a prompt, introduce unintended side effects, or generate code that is syntactically correct but logically flawed. Change review allows you to:

*   **Verify correctness:** Ensure the code does what it's supposed to do.
*   **Catch regressions:** Make sure new code hasn't broken existing functionality.
*   **Maintain code style:** Align generated code with your project's coding standards.
*   **Identify security vulnerabilities:** Spot potential security flaws.
*   **Understand the changes:** Gain insight into how the agent solved the problem.

## Practical Scenario: Adding a User Authentication Check

Imagine you've asked an agent to implement a basic user authentication check before accessing a protected API endpoint. The agent generates a diff. Your task is to review it.

**Generated Diff (Simplified):**

```diff
--- a/api/routes.js
+++ b/api/routes.js
@@ -1,5 +1,9 @@
 const express = require('express');
 const router = express.Router();
+
+function isAuthenticated(req, res, next) {
+  // In a real app, this would check a token or session
+  next(); // For now, always allow access
+}
 
 router.get('/public', (req, res) => {
   res.send('Public data');
@@ -7,7 +11,7 @@
 
 router.get('/protected', (req, res) => {
   res.send('Protected data');
-});
+}); // Issue: isAuthenticated not applied here
 
 module.exports = router;
```

**Your Review Process:**

1.  **Read the Description:** The agent was supposed to add an authentication check.
2.  **Examine the `+` lines:** New code is added. A `isAuthenticated` function is defined. This looks reasonable as a placeholder.
3.  **Examine the `-` lines:** No lines were removed, which is expected for adding a new feature.
4.  **Identify Changes to Existing Code:** The `router.get('/protected', ...)` block has a new closing parenthesis and semicolon. This is where the critical review happens.
5.  **Compare to Requirements:** The `isAuthenticated` function is defined, but it's **not being used** in the `/protected` route handler. The original code is essentially replaced with a slightly modified but functionally identical route that bypasses the intended authentication.

**Your Action:** You would reject this change or ask the agent to revise it, pointing out that the `isAuthenticated` middleware needs to be applied to the protected route.

## Practice Task

Consider the following hypothetical agent-generated diff for a Python function that sorts a list of numbers.

```diff
--- a/sort_utils.py
+++ b/sort_utils.py
@@ -1,4 +1,4 @@
 def bubble_sort(data):
     n = len(data)
     for i in range(n):
-        for j in range(0, n-i-1):
+        for j in range(0, n-i): # Potential off-by-one error
             if data[j] > data[j+1]:
                 data[j], data[j+1] = data[j+1], data[j]
     return data
```

Inspect this diff. What is the potential issue, and what would you do?
