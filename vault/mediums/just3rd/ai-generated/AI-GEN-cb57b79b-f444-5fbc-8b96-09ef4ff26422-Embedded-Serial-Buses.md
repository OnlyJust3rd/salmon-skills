# Embedded Serial Buses: SPI, I2C, and One-Wire

This lesson focuses on understanding the typical roles of SPI, I2C, and One-Wire serial buses in embedded device communication. By grasping these protocols, you'll be better equipped to analyze specific hardware interfaces and make informed decisions about which interface to choose for a given embedded scenario.

## Understanding the Need for Embedded Serial Buses

In embedded systems, microcontrollers (MCUs) often need to communicate with various peripheral devices like sensors, memory chips, displays, and other MCUs. These peripherals might require a dedicated communication channel. Serial buses offer a way to transmit data bit by bit over a small number of wires, making them efficient for connecting multiple devices without excessive cabling. SPI, I2C, and One-Wire are popular choices for these internal communication needs within an embedded system.

## SPI (Serial Peripheral Interface)

SPI is a synchronous serial communication interface protocol developed by Motorola. It's known for its high speed and full-duplex communication, meaning data can be sent and received simultaneously.

### Key Components of SPI:

*   **Master/Slave Architecture:** One device (the master) controls the communication, initiating data transfers and dictating the clock signal. Other devices (slaves) respond to the master.
*   **Four Wires:**
    *   **SCLK (Serial Clock):** Generated by the master, synchronizes data transfer.
    *   **MOSI (Master Out, Slave In):** Data sent from the master to the slave.
    *   **MISO (Master In, Slave Out):** Data sent from the slave to the master.
    *   **SS/CS (Slave Select / Chip Select):** An active-low signal used by the master to select which slave device it wants to communicate with. Multiple slaves can be connected, each with its own SS line from the master.

### Typical Role of SPI:

SPI is ideal for high-speed, relatively short-distance communication within an embedded system where a dedicated connection is needed.

*   **High-speed sensors:** Like accelerometers or gyroscopes where rapid data acquisition is crucial.
*   **Flash memory or SD card interfaces:** For quick program loading or data storage.
*   **Digital-to-Analog Converters (DACs) and Analog-to-Digital Converters (ADCs):** For efficient data conversion.
*   **Display drivers:** To update screen content quickly.

### SPI Characteristics:

| Feature           | Description                                                |
| :---------------- | :--------------------------------------------------------- |
| **Speed**         | High (often tens of MHz)                                   |
| **Duplex**        | Full-duplex                                                |
| **Addressing**    | Uses dedicated Slave Select (SS) lines for each slave.     |
| **Complexity**    | Relatively simple to implement in hardware.                |
| **Power**         | Can consume more power due to higher speeds.               |
| **Number of Wires** | Minimum 3 (if MISO/MOSI are bidirectional or combined), typically 4. |

## I2C (Inter-Integrated Circuit)

I2C is a synchronous, multi-master, multi-slave serial communication bus invented by Philips Semiconductors. It's designed for short-distance communication, typically on a single circuit board.

### Key Components of I2C:

*   **Master/Slave Architecture:** Similar to SPI, a master initiates communication. However, I2C supports multiple masters, although only one can be in control at any given time.
*   **Two Wires:**
    *   **SCL (Serial Clock):** Generated by the master, synchronizes data transfer.
    *   **SDA (Serial Data):** Carries both data and acknowledges signals.

### Addressing:

Each device on the I2C bus has a unique 7-bit or 10-bit address. The master sends the address of the target slave before sending or receiving data. This shared bus approach means you can connect many devices (up to 127 with 7-bit addressing) using only two wires.

### Typical Role of I2C:

I2C is excellent for connecting a moderate number of devices at moderate speeds on the same board, especially when wire count is a concern.

*   **Small sensors:** Temperature sensors, humidity sensors, light sensors.
*   **Real-time clocks (RTCs):** For keeping track of time.
*   **EEPROM or small serial flash memory:** For configuration data.
*   **I/O expanders:** To add more input/output pins to an MCU.
*   **On-board management controllers:** In complex systems.

### I2C Characteristics:

| Feature           | Description                                                 |
| :---------------- | :---------------------------------------------------------- |
| **Speed**         | Moderate (Standard: 100 kHz, Fast: 400 kHz, High-speed: 1-3.4 MHz) |
| **Duplex**        | Half-duplex (data sent and received sequentially, not simultaneously) |
| **Addressing**    | Uses unique 7-bit or 10-bit addresses on a shared bus.      |
| **Complexity**    | More complex arbitration and ACK handling than SPI.           |
| **Power**         | Generally lower power consumption than SPI.                 |
| **Number of Wires** | 2 (SCL and SDA)                                             |

## One-Wire

One-Wire is a serial communication protocol developed by Dallas Semiconductor (now Maxim Integrated). As the name suggests, it requires only a single data wire for communication, plus a ground connection.

### Key Components of One-Wire:

*   **Single Data Wire:** This wire carries both data and power (in some configurations) and is also used for timing.
*   **Master/Slave:** A master device initiates communication.
*   **Unique ROM IDs:** Each One-Wire device has a unique 64-bit ROM ID programmed into it. The master can use these IDs to identify and communicate with specific devices on the bus.

### Typical Role of One-Wire:

One-Wire is best suited for applications where extreme simplicity in wiring is paramount, and the data transfer speeds are not critical.

*   **Temperature sensors:** Like the DS18B20, which are very common and easy to integrate.
*   **Identification tags/keys:** For simple authentication.
*   **Memory devices:** For storing small amounts of configuration data.
*   **Simple switch or status reporting:** Where low bandwidth is sufficient.

### One-Wire Characteristics:

| Feature           | Description                                                   |
| :---------------- | :------------------------------------------------------------ |
| **Speed**         | Slow (typically 15-125 kbps)                                  |
| **Duplex**        | Half-duplex                                                   |
| **Addressing**    | Uses unique ROM IDs to address devices.                       |
| **Complexity**    | Protocol requires precise timing. Hardware support can vary.  |
| **Power**         | Can use parasitic power from the data line, reducing pin count. |
| **Number of Wires** | 1 data wire + ground (effectively 2 wires total).             |

## Conclusion

SPI, I2C, and One-Wire are fundamental embedded serial buses, each with its strengths. SPI excels in speed and full-duplex communication for dedicated device connections. I2C offers a balance of moderate speed, multi-device addressing, and a low pin count for board-level communication. One-Wire prioritizes extreme simplicity with its single-wire design, making it suitable for low-bandwidth, simple sensor applications. Understanding these differences is crucial for analyzing specific hardware interfaces and selecting the most appropriate one for your embedded device scenarios.
